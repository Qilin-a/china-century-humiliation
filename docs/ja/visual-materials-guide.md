# 視覚資料ガイド

このガイドは、中国の屈辱の世紀歴史文書プロジェクトに歴史的画像、地図、写真、その他の視覚資料を追加するための推奨事項を提供します。

---

## 概要

視覚資料は歴史的出来事の理解を大幅に向上させることができます。このガイドでは以下を概説します：
- 含めるべき視覚資料の種類
- 著作権とライセンスに関する考慮事項
- 推奨される情報源
- ファイルの整理と命名規則
- 帰属表示の要件

## 著作権とライセンス

### パブリックドメイン資料

**一般的に安全に使用できるもの**：
- 1928年以前に発行された作品（ほとんどの司法管轄区で）
- 中国政府文書（特定の法律を確認）
- 著者が70年以上前に亡くなった作品
- 明示的にパブリックドメインとマークされた作品

**使用前に確認**：
- 関連する司法管轄区での著作権状態を確認
- 発行日を確認
- 該当する場合、著者の死亡日を確認

### クリエイティブ・コモンズ資料

**受け入れ可能なライセンス**：
- CC0（パブリックドメイン）
- CC BY（表示）
- CC BY-SA（表示-継承）- 当プロジェクトのライセンスと互換性あり

**避けるべきもの**：
- CC BY-NC（表示-非営利）
- CC BY-ND（表示-改変禁止）
- すべての権利予約

### フェアユースの考慮事項

**教育的使用**：
- このプロジェクトは教育的性質を持つ
- 必要な範囲でのみ使用
- 適切な帰属表示を提供
- 変形的使用が望ましい

**ベストプラクティス**：
- 可能な限り許可を求める
- 利用可能な場合はパブリックドメインを使用
- 常に明確な帰属表示を提供

### 命名規則

**形式**：`説明-年.拡張子`

**例**：
- `lin-zexu-portrait.jpg`
- `treaty-nanking-1842.jpg`
- `battle-of-yellow-sea-1894.png`
- `may-fourth-protest-1919.jpg`

**ルール**：
- 小文字を使用
- ハイフンで単語を区切る
- 関連する場合は年を含める
- 説明的だが簡潔に
- 標準のファイル拡張子を使用

## 画像仕様

### 技術要件

**解像度**：
- 最小：最長辺1200ピクセル
- 推奨：アーカイブ品質には2000ピクセル以上
- 地図：詳細を示すためにより高解像度

**ファイル形式**：
- 写真：JPEG（品質85-95%）
- グラフィック：透明性のためPNG
- 図表：可能な限りSVG
- 文書：高解像度JPEGまたはPDF

**ファイルサイズ**：
- 品質とファイルサイズのバランス
- ウェブ表示用に最適化
- アーカイブ用に元データを保持

### アクセシビリティ

**代替テキスト**：
- 説明的な代替テキストを提供
- 表示内容を説明
- 関連する文脈を含める
- 簡潔だが情報豊富に

**キャプション**：
- 日付と場所
- 簡単な説明
- 歴史的文脈
- 情報源の帰属表示

## 帰属表示要件

### 必要情報

各画像について提供：

1. **タイトル/説明**：画像が示すもの
2. **日付**：作成/撮影時期
3. **作成者**：写真家、芸術家、または組織
4. **情報源**：入手先
5. **ライセンス**：著作権状態またはライセンスタイプ
6. **URL**：オンラインの場合は元のリンク

### 帰属表示の形式

**Markdownでの記述**：
```markdown
![説明](path/to/image.jpg)

*画像：[タイトル]。[日付]。[作成者]。出典：[機関/アーカイブ]。[ライセンス]。[URL]*
```

**例**：
```markdown
![林則徐の肖像](images/photos/people/lin-zexu.jpg)

*画像：林則徐の肖像。1840年頃。作者不詳。出典：ウィキメディア・コモンズ。パブリックドメイン。https://commons.wikimedia.org/wiki/File:Lin_Zexu.jpg*
```

## 法的および倫理的考慮事項

### センシティブなコンテンツ

**戦争残虐行為**：
- 歴史的正確性のために含める
- コンテンツ警告を提供
- グラフィック画像の使用に慎重
- 教育的価値に焦点

**文化的配慮**：
- すべての視点を尊重
- ステレオタイプ的な画像を避ける
- 文脈を提供
- 論争を認識

### 許可

**許可を求めるべき時**：
- 最近の写真（1928年以降）
- 著作権状態が不明
- 商業アーカイブ
- 私的コレクション

**リクエスト方法**：
- 自己とプロジェクトを紹介
- 教育目的を説明
- 意図する使用を明示
- 帰属表示を提供
- 必要に応じて料金支払いの準備
